# /intake - Product Discovery (Levine/Cagan Style)

## Overview
Inicia o processo de descoberta com o **PM Product** (@.cursor/agents/pm_product.md). Foco total no problema, não na solução.

## Protocolo de Execução
### 1. Input Obrigatório
- Uma "dor" ou ideia bruta do usuário.

### 2. Skills Requeridas
- `@.cursor/skills/product-intake/SKILL.md` (Template de Spec)

### 2. Pré-condição (Gates)
- **MCP Plane**: Conexão ativa necessária para criação de Epic.
- **Fase**: Descoberta.

### 3. Ações
1. **Entrevista**: PM faz 5-8 perguntas "Why" para chegar à raiz do problema.
2. **Definição**: Mapeamento de User Stories focadas em resultados (outcomes).
3. **Sincronização**: Cria a descrição do Epic diretamente no Plane.so via MCP.

### 4. Output Obrigatório
- Relatório de descoberta registrado no Plane.so com problema validado e critérios de aceite.

### 5. Próximo Estado
- Pronto para `/plan`.
